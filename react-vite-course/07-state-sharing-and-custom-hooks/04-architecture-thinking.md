# Architecture Thinking for React Apps

## Ask these questions
- What data lives where?
- Which component owns the state?
- Which parts are reusable?
- What should be local state?
- What should be shared state?
- What happens on loading/error/empty?

## Good architecture traits
- clear folder structure
- components with one clear responsibility
- reusable pieces
- predictable data flow
- fewer giant files

## Bad architecture traits
- one file doing everything
- unclear naming
- duplicated logic everywhere
- state stored in random places
