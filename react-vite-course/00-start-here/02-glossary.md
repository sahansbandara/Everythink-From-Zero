# Essential Glossary

| Term | Meaning |
|---|---|
| HTML | Markup language for structure |
| CSS | Style language for appearance |
| JavaScript | Programming language for behavior and logic |
| Frontend | The part users see in the browser |
| Backend | Server-side logic and data processing |
| Database | Stores persistent data |
| Node.js | JavaScript runtime outside the browser |
| npm | Node package manager |
| Package | Reusable code you can install |
| Dependency | Package your project needs |
| React | JavaScript library for building UI |
| Vite | Frontend dev server and build tool |
| JSX | HTML-like syntax inside React code |
| Component | Reusable UI piece |
| Props | Data passed into a component |
| State | Data that can change inside a component |
| Hook | Special React function like `useState` |
| Routing | Moving between pages/views in an app |
| API | System that allows programs to communicate |
| Build | Optimized production output |
| `node_modules` | Folder containing installed packages |
| `package.json` | Project metadata, scripts, dependencies |
| `package-lock.json` | Exact dependency versions |

## Fast self-test
If someone asks “What is React?” your answer should not be:
> a language

It should be:
> a JavaScript library for building user interfaces
