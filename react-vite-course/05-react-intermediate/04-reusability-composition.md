# Reusability and Composition

## Composition
Building larger UIs by combining smaller components.

Example:
- `Card`
- `CardHeader`
- `CardBody`
- `Button`

## Why composition matters
It keeps your code:
- modular
- reusable
- easier to scale

## Bad approach
Huge giant components doing everything.

## Better approach
Split by responsibility:
- layout
- data
- presentation
- interaction
