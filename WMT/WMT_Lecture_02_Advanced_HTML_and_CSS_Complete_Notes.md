A CSS rule set consists of a selector and a declaration block:

The declaration block contains one or more declarations separated by

## Page 18 — Specifying Color by value

### Original Slide Content

Specifying Color by value
The format of an RGB value in hexadecimal notation is a #
immediately followed by either three or six hexadecimal characters.
A general colour is specified in terms of the three primary colours red,
blue and green.
Each primary colour is defined as a one or two digit Hexadecimal
number that representing the strength of that primary colour.
The three-digit RGB notation (#rgb) is converted into six-digit form
(#rrggbb) by replicating digits, not by adding zeros.
For example, #fb0 expands to #ffbb00, and #fff expands to #ffffff
In this specification hexadecimal 00 means 0% of the colour and
hexadecimal ff means 100% of the colour.

### Visual Explanation

No major visual content. The slide explains hexadecimal RGB color notation.

### Explanation

This slide explains styling of visible HTML content such as colors, lists, navigation bars, and tables. These topics often appear as short practical code-writing questions.

### Exam Tip

Medium priority. Practice writing small HTML/CSS snippets for colors, lists, navigation bars, and tables.

### Common Mistake

Three-digit hex expands by duplicating each digit: #fb0 becomes #ffbb00, not #f0b000.

## Page 20 — Functional RGB color values

### Original Slide Content

Functional RGB color values
The format of an RGB value in the functional notation is rgb(
followed by a comma-separated list of three numerical values (either
three integer values or three percentage values) followed by ).
The integer value 255 corresponds to 100%, and to F or FF in the
hexadecimal notation: rgb(255,255,255) = rgb(100%,100%,100%) =
#FFF = #FFFFFF.
White space characters are allowed around the numerical values.

# Full Lecture Summary

Lecture 02 explains how advanced HTML and CSS are used to create styled, structured, interactive web pages. The lecture begins with CSS syntax, showing that a CSS rule contains a selector and declarations. It compares inline, embedded, and external CSS. It then explains selectors: element, grouped, descendant/context, class, ID, combined, universal, and pseudo-class selectors. The lecture moves into color values, list styling, navigation bars, and HTML table styling. After that, it introduces the CSS box model and explains how displayed size is calculated using content width, padding, border, and margin. The lecture then covers images, background images, image maps, hyperlinks, URLs, email links, redirection, iframes, special characters, and styling links. The layout section explains positioning, floats, fluid vs fixed layouts, float drop, box-sizing, and clear. The final major section covers HTML forms: form tags, form attributes, text inputs, labels, textarea, password fields, radio buttons, checkboxes, dropdowns, scrolling lists, datalist, HTML5 input types, useful attributes, fieldset/legend, reset, submit, form submission format, styling form inputs, and emailing form data.

# Likely Exam Questions and Short Answers

## Question 1
Define CSS.

**Expected Answer:**
CSS is Cascading Style Sheets. It works with HTML to define how content is presented, including colors, fonts, layout, spacing, and animations.

## Question 2
What are the three ways to insert CSS?

**Expected Answer:**
Inline style, embedded/internal stylesheet, and external stylesheet.

## Question 3
Why is external CSS preferred for final websites?

**Expected Answer:**
It separates presentation from HTML, can be reused across many pages, and updates all linked pages when the CSS file changes.

## Question 4
Write the structure of a CSS rule.

**Expected Answer:**
A CSS rule has a selector and a declaration block: selector { property: value; }. Declarations are separated by semicolons.

## Question 5
Differentiate class selector and ID selector.

**Expected Answer:**
A class selector uses dot and can be used by many elements. An ID selector uses hash and should target one unique element.

## Question 6
What does p em select?

**Expected Answer:**
It selects em elements that are nested inside p elements.

## Question 7
What does div.important h2 select?

**Expected Answer:**
It selects h2 elements inside a div element that has class important.

## Question 8
What is the CSS box model?

**Expected Answer:**
It is the model where every element is treated as a rectangular box with content, optional padding, border, and margin.

## Question 9
How do you calculate total width of an element?

**Expected Answer:**
Total width = content width + left/right padding + left/right border + left/right margin.

## Question 10
What is float drop?

**Expected Answer:**
Float drop happens when floated columns do not have enough horizontal space, so one column drops below another.

## Question 11
Give two solutions for float drop.

**Expected Answer:**
Reduce column widths or use CSS box-sizing:border-box so padding and borders are included within the declared width.

## Question 12
What is the clear property?

**Expected Answer:**
clear prevents an element from wrapping around floated elements. Common option: clear:both.

## Question 13
Which tag displays an image?

**Expected Answer:**
The img tag displays an image.

## Question 14
What are important img attributes?

**Expected Answer:**
src identifies image file path, and alt provides alternative text for accessibility/fallback.

## Question 15
What is an image map?

**Expected Answer:**
An image map is an image with clickable regions defined using map and area tags.

## Question 16
What is a URL?

**Expected Answer:**
A Uniform Resource Locator specifies the location of a file/resource on a remote machine.

## Question 17
What is an iframe?

**Expected Answer:**
An iframe displays a web page within another web page.

## Question 18
Why are HTML special characters needed?

**Expected Answer:**
Characters such as < have special meaning in HTML, so entities are used to display them as text.

## Question 19
What is the purpose of a form?

**Expected Answer:**
A form collects user input and submits it to a server-side script or email target for processing.

## Question 20
What are form action and method attributes?

**Expected Answer:**
action gives the target URL/script/email. method specifies how data is sent, such as GET or POST.

## Question 21
Which method is default if form method is omitted?

**Expected Answer:**
GET is the default method in browsers.

## Question 22
Why are labels useful?

**Expected Answer:**
Labels provide captions, increase clickable area, and help screen readers identify fields.

## Question 23
Difference between implicit and explicit labels?

**Expected Answer:**
Implicit labels wrap the input. Explicit labels are separate and connect to the input using for matching the input id.

## Question 24
How do radio buttons work as a group?

**Expected Answer:**
Radio buttons belong to the same group when they share the same name attribute.

## Question 25
Difference between radio buttons and checkboxes?

**Expected Answer:**
Radio buttons allow one selected option per group; checkboxes allow multiple independent selections.

## Question 26
What does datalist do?

**Expected Answer:**
It provides suggested options for a text input.

## Question 27
What is datetime-local input?

**Expected Answer:**
It lets the user enter local date and time using yyyy-mm-ddThh:mm format.

## Question 28
What is a hidden field?

**Expected Answer:**
A hidden field submits a name-value pair without showing a visible control on the form.

## Question 29
What does required do?

**Expected Answer:**
It makes the field mandatory before form submission.

## Question 30
How is form data submitted when several elements exist?

**Expected Answer:**
Individual name=value pairs are concatenated using &, for example firstName=Jane&lastName=Smith&button=Send.

# Common Mistakes to Avoid

- Writing CSS code without semicolons between declarations.
- Confusing embedded CSS with external CSS.
- Putting <style> tags inside an external .css file.
- Confusing class selectors (.) and ID selectors (#).
- Reusing the same ID many times in one page.
- Forgetting that padding, border, and margin increase displayed width in the normal box model.
- Saying float drop happens randomly; it happens because total width exceeds available space.
- Confusing <tr>, <td>, and <th>.
- Forgetting alt attribute for images.
- Confusing href links and src image paths.
- Using placeholder instead of proper label.
- Using different name values for radio buttons that should be in one group.
- Assuming plain button automatically submits a form.
- Forgetting that form controls need name attributes for submitted data.
- Using GET for sensitive form information.
- Thinking mailto form submission is reliable for real applications.

# Final One-Page Revision Notes

## Core CSS
- CSS controls presentation: colors, fonts, spacing, layout, animations.
- Rule structure: `selector { property: value; }`.
- CSS insertion: inline, embedded, external.
- Best practice: use external CSS for real websites.
- Selectors: `p`, `h1,h2`, `p em`, `.class`, `#id`, `div.class`, `*`, `a:hover`.

## Box Model
```text
Margin
└── Border
    └── Padding
        └── Content
```
- Total width = content width + left/right padding + left/right border + left/right margin.
- `box-sizing: border-box` includes padding and border inside declared width.

## Images and Links
- Image syntax: `<img src="file.jpg" alt="description">`.
- Image map: `<img usemap="#map">` + `<map>` + `<area>`.
- Link syntax: `<a href="page.html">Text</a>`.
- Email link: `mailto:`. Redirect: meta refresh.
- Link states: `a:link`, `a:visited`, `a:hover`, `a:active`.

## Forms
- Form syntax: `<form action="..." method="post">...</form>`.
- Default method: GET.
- Text: `input type="text"`; password: `password`; email: `email`; number: `number`; date: `date`; color: `color`.
- Radio group: same `name`; checkbox: independent multiple selections.
- Explicit label: `<label for="x">` + input `id="x"`.
- Required, readonly, autofocus, placeholder are useful attributes.
- Submitted format: `name=value&name=value`.

## Exam Keywords
- selector, declaration, property, value
- inline / embedded / external CSS
- class vs ID
- content, padding, border, margin
- float drop, box-sizing, clear
- src, alt, href, URL, iframe
- form, action, method, input, label, submit, reset
